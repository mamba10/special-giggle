a project for European control room of a leading energy manufacturer. The goal of the project was to create a centralized system for monitoring actual production and tracking failures of ca. 3000 wind turbines located in 200 farms in 10 countries. In this system we have integrated multiple technologies: the component that retrieves real-time data from external data sources is written in C# and deployed as a windows service. It communicates with the web application via a message queue. The web application itself is written in Scala and uses several Scala and Java frameworks and tools, such as Akka, Spring, Maven, Mybatis-scala or ScalaTest. The data is exposed to clients using websockets (we use atmosphere framework for that) and REST web services. Finally the client side is written using AngularJS.

Responsibilities:
requirement analysis, architecture design, development, project leadership
Technologies:
Scala, Akka, Spring, WebSockets, AngularJS, C#

Polish Component of Schengen Information System (SIS1+)

Schengen Information System (SIS) is a database system used by European countries to record and share information about persons and objects (vehicles, firearms, documents, etc.) of interest. The goal of the project was to create an integration layer between SIS and information systems of Polish national institutions such as The National Police, The Border Guard and others. The integration was necessary for Poland to join Schengen Agreement in December 2007. All systems communicated using SOAP web services. The integration layer was mainly responsible for translating and routing messages created using different schemas.

Responsibilities:
development
Technologies:
Java, Java EE, JBoss AS, Oracle DB, SOAP

Polish Component of Schengen Information System (SISII)

a second technical version of Schengen Information System. The goal of the project to was to integrate systems that together formed Polish Component of SISII. Proprietary Enterprise Service Bus was used for integrating systems provided by various vendors and created in various technologies.

Responsibilities:
system integration, development
Technologies:
Java, BEA/Oracle WebLogic, Apache Tomcat, Oracle DB, ESB, SOAP, XML Digital Signature, C++

Employee Satisfaction Survey

a web portal for conducting flexible employee satisfaction surveys, targeted to small and medium enterprises, developed for a human resources startup.

Responsibilities:
architecture design, development, deployment
Technologies:
Scala, Slick, Play, AngularJS, Bootstrap, MySQL

Repository and hosting platform

a system that allowed hosting many repositories for many customers. Each repository had a completely configurable meta-data model and could be used for storing any kind of digital objects. Object representations (including web portal pages) were configurable per repository and per object type. Many import and export mechanisms were implemented as well as transformations between most popular meta-data formats. The system also included flexible search mechanism and fine-grained security. Extensive REST API was created that provided access to functionality of the platform.

Responsibilities:
team leading, architecture design, development
Technologies:
Java, Spring, Spring Security, Apache Tomcat, PostgreSQL, Neo4J, Elasticsearch, HDFS, REST

Rule Generation Engine for Content Classification

a system that exposes a semantic model stored in an ontology, enriched with additional classification-related data (such as how an occurrence of a term of some type should affect the classification result depending on the document section in which the occurrence was found), to a document classification server. The classification-related data is supplied as user-defined templates. Simple natural language processing is applied to the terms before the classification rules are created.

Responsibilities:
design of architecture and algorithms, development, creation of automated testing framework
Technologies:
Python, Spring Python, SQLite

Backend system for mobile application for salesmen in global electric corporation

a system responsible for storing data and sharing it between salespeople and generating reports. It integrates with various other corporate backend systems, including single sign-on, external authorization and document repository.

Responsibilities:
REST API design, architecture design, development
Technologies:
Java, Spring, Spring Security, Apache Tomcat, Oracle DB, REST

Recommendation Engine for Real Estate Market

a system that analyzes real estate sale and rental offers in major Polish cities and creates investment recommendations based on expected return of investment of a property depending on criteria such as location, size, floor etc.

Responsibilities:
design, development, deployment
Technologies:
Python, Django, Scrapy, MySQL, Google App Engine, MS Azure
